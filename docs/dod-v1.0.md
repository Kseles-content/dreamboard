# DreamBoard — Definition of Done (v1.0)

## 1) Функциональные требования

### 1.1 Авторизация и доступ
- Логин/регистрация по email + name (через `/v1/auth/login`), выдача `accessToken` и `refreshToken`.
- Обновление access-токена через refresh flow (`/v1/auth/refresh`) с ротацией refresh-токена.
- Логаут и отзыв токенов (`/v1/auth/logout`, `/v1/auth/revoke-token`, `/v1/auth/revoke-all`).
- Защита приватных эндпоинтов: без bearer-токена возвращается `401` с единым error-envelope.

### 1.2 CRUD досок
- Создание доски (пустой) `/v1/boards`.
- Получение списка/деталей досок `/v1/boards`, `/v1/boards/:id`.
- Обновление доски `/v1/boards/:id`.
- Удаление доски (soft-delete) `/v1/boards/:id`.
- Пины и recent-доски (`/v1/boards/:id/pin`, `/v1/boards/recent`).

### 1.3 CRUD карточек
- Создание карточки `/v1/boards/:id/cards`.
- Редактирование карточки `/v1/boards/:id/cards/:cardId`.
- Удаление карточки `/v1/boards/:id/cards/:cardId`.
- Получение списка карточек `/v1/boards/:id/cards`.

### 1.4 Шаринг
- Создание share-ссылки `/v1/boards/:id/share-links`.
- Просмотр и отзыв share-ссылок `/v1/boards/:id/share-links`, `DELETE /v1/boards/:id/share-links/:linkId`.
- Публичный read-only просмотр `/v1/share/:token`.

### 1.5 Экспорт
- Экспорт доски в PNG/JPG доступен в web UI (кнопки `Export PNG` / `Export JPG`, клиентский рендер canvas).

### 1.6 Поиск и фильтрация
- Поиск по названию (`query`) в `/v1/boards`.
- Фильтр по pinned (`pinned=true/false`).
- Фильтр по времени обновления (`updatedSince`).
- Стабильная cursor-пагинация (`limit`, `cursor`).

### 1.7 Создание из шаблона
- Получение списка шаблонов `/v1/templates`.
- Создание доски из шаблона `/v1/boards/from-template`.

### 1.8 Resume flow
- При открытии доски обновляется `lastOpenedAt`.
- Экран recent корректно показывает недавно открытые доски (`/v1/boards/recent`).

---

## 2) Нефункциональные требования

### 2.1 Скорость загрузки / отклика
- `/health` доступен и отвечает `{"status":"ok"}`.
- Ключевые API-операции (auth, board/card CRUD, share) проходят без таймаутов в стандартном dev-окружении.
- Пагинация списка досок не деградирует при росте количества досок (проверяется cursor flow).

### 2.2 Обработка ошибок
- Единый формат error-envelope для ошибок API (`code`, `message`, `requestId`).
- Для частых ошибок используются машинно-читаемые коды (`UNAUTHORIZED`, `NOT_FOUND`, `BOARD_LIMIT_REACHED`, `SHARE_LINK_NOT_FOUND`, и т.д.).
- Некорректные входные данные валидируются на API-слое (ValidationPipe + DTO).

### 2.3 Надёжность и безопасность
- JWT для защищённых эндпоинтов.
- Refresh-token rotation и отзыв токенов.
- Публичные share-ссылки ограничены read-only доступом и могут быть отозваны владельцем.

## 3) Критерии приёмки перед передачей пользователю

Перед handoff версии v1.0 должны быть одновременно выполнены:

1. **Документация DoD актуальна**
   - Файл `docs/dod-v1.0.md` существует и отражает текущий scope v1.0.

2. **Нет открытых P1 багов**
   - В GitHub нет open issues с приоритетом `P1`/`critical`.

3. **Ключевые сценарии подтверждены**
   - Регистрация/логин.
   - Создание доски (пустой и из шаблона).
   - Создание/редактирование/удаление карточек.
   - Share-ссылка + публичный просмотр.
   - Экспорт PNG/JPG.
   - Поиск и фильтрация досок.
   - Resume flow (continue where you left off).

4. **Базовый регрессионный прогон зелёный**
   - `npm run test:e2e` проходит без падений.
